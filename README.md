# ai-selenium-automation-claude-sdet
Selenium + CI/CD + Claude AI usage

I integrated Claude Opus into this Selenium framework to dynamically generate validation points and edge-case scenarios during test execution. This shifts traditional, static automation toward AI-assisted testing, helping improve test coverage and identify gaps early, especially in complex UI workflows.
Note: Due to confidentiality, the actual AI prompts have been omitted. The current implementation includes placeholder code to demonstrate how Claude integration is structured within the framework.

This framework demonstrates:
- Selenium WebDriver automation using Page Object Model (POM)
- AI-assisted test design using Claude Opus
- CI/CD integration (Azure DevOps / GitHub Actions)
- API + UI automation coverage
- Log analysis using GenAI

AI capabilities:
- Test case generation from requirements
- Edge-case expansion
- Automation script suggestions
- Failure log analysis

Tech Stack:
.NET
- Selenium WebDriver
- NUnit
- AI (Claude Opus)
- CI/CD pipelines
